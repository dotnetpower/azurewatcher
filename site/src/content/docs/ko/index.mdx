## Forward Deployed Agents가 이벤트를 해소하는 방식

<p class="hero-note">신호부터 감사까지, 하나의 제어 루프. 모든 이벤트는 정규화되고, 신뢰도로 라우팅되고, 리스크로 게이팅됩니다 - 그리고 언어 모델은 결정론 티어가 의도적으로 넘긴 소수의 모호한 케이스에만 쓰입니다.</p>

## 자율성이 여러분에게 주는 것

<ul class="impact-list">
  <li>
    <span class="impact-tick" aria-hidden="true">↳</span>
    <strong>이벤트의 ~70-80%가 결정론적으로 해소.</strong> 규칙 · 정책 ·
    체크리스트가 반복 가능한 다수를 결정. 모델 호출 없음.
  </li>
  <li>
    <span class="impact-tick" aria-hidden="true">↳</span>
    <strong>~15-20%는 경량 패턴 재사용(T1)으로 해소.</strong> 과거 해소된
    인시던트와의 임베딩 유사도, 소형 분류기, frontier 모델 없음.
  </li>
  <li>
    <span class="impact-tick" aria-hidden="true">↳</span>
    <strong>추론 티어(T2)에 도달하는 것은 ~5-10%뿐.</strong> mixed-model
    교차 검증 + 결정론 verifier - 모델은 제안하고, verifier가 처분합니다.
  </li>
  <li>
    <span class="impact-tick" aria-hidden="true">↳</span>
    <strong>모든 자율 액션은 네 불변식을 지닌다.</strong> stop-condition ·
    롤백 경로 · blast-radius 제한 · 실행 기록 항목. 넷 중 하나라도 없으면
    액션은 AUTO가 아니라 HIL로 갑니다.
  </li>
  <li>
    <span class="impact-tick" aria-hidden="true">↳</span>
    <strong>새 능력은 shadow로 시작하며, 결코 깜짝 등장하지 않는다.</strong>
    Enforce로의 승격은 Phase 0 기준선 대비 별도의 측정 가능한 게이트입니다.
  </li>
  <li>
    <span class="impact-tick" aria-hidden="true">↳</span>
    <em>위 수치는 모두 설계 목표이지 측정된 결과가 아닙니다.</em>
    Forward Deployed Agents는 짝을 이룬 기준선 측정 없이 배수를 주장하지 않습니다.
  </li>
</ul>

## 하나의 코어, 세 도메인

<p class="hero-note">세 도메인 모두 같은 이벤트 기반 · 리스크 게이트 코어를 공유한다. 다른 것은 각 도메인이 로드하는 규칙과 액션뿐이다.</p>

<CardGrid stagger>
  <Card title="Change Safety · Phase 1" icon="pencil">
    규칙 카탈로그, T0 정책 게이트, remediation PR. Change 게이트는 먼저 shadow로 도입한 뒤 enforce로 승격한다.
  </Card>
  <Card title="Resilience · Phase 3" icon="rocket">
    스케줄링된 회복력 검증, DB DR 훈련, blast-radius 제한 카오스 실험 - 항상 stop-condition과 롤백 경로를 갖춘다.
  </Card>
  <Card title="Cost Governance · Phase 3" icon="approve-check">
    비용 이상 탐지, 라이트사이징 PR, 리소스 그룹별 예산 가드레일. 리스크 분류에 따라 auto vs HIL로 나뉜다.
  </Card>
</CardGrid>

## 에이전트 조직을 만나보세요

<p class="hero-note">15개의 이름을 가진 에이전트가 하나의 조직으로 제어 루프를 운영합니다 - 각자 역할을 맡고 스키마 검증된 이벤트 버스로 인계합니다. 은유가 아니라 조직도입니다: 판단자는 실행하지 않으며, 모든 액션은 서로 다른 에이전트가 개시, 판단, 승인, 실행, 감사합니다.</p>

<CardGrid>
  <Card title="Odin - 지휘" icon="star">
    마스터 플래너. 버티컬 간 중재와 판정 확정 전 최종 타이브레이크.
  </Card>
  <Card title="판단과 실행" icon="approve-check">
    **Forseti**가 판단(auto / HIL / deny), **Thor**가 디스패치·실행, **Var**가 사람 승인, **Vidar**가 롤백·DR 페일오버를 담당합니다. 어떤 에이전트도 판단과 실행을 겸하지 않습니다.
  </Card>
  <Card title="감지" icon="magnifier">
    **Huginn**이 이벤트를 수집·상관하고, **Heimdall**이 상태와 드리프트를 관측합니다. 결정론 우선 - 핫패스에 LLM은 없습니다.
  </Card>
  <Card title="대화와 감사" icon="comment">
    **Bragi**가 운영자 질문을 도구 호출로 번역하고, **Saga**가 append-only 감사 로그를 쓰고 인계를 에스컬레이션합니다.
  </Card>
  <Card title="거버넌스" icon="setting">
    **Mimir**가 룰 카탈로그를 관리하고, **Norns**가 결과에서 학습하며, **Muninn**이 장기 기억을 보유합니다.
  </Card>
  <Card title="도메인 전문가" icon="puzzle">
    **Njord**(비용), **Freyr**(용량), **Loki**(카오스)가 도메인 분석으로 판단자를 보조합니다 - 직접 실행하지 않습니다.
  </Card>
</CardGrid>

## 배포 단계 (Delivery Phases)

<p class="hero-note">설계상 순차적입니다 - 각 단계는 이전 단계 위에 쌓입니다. 참조 문서를 먼저 읽고 단계 문서를 순서대로 따라가세요.</p>

<ol class="phase-timeline">
  <li>
    <span class="phase-marker" aria-hidden="true" />
    <div class="phase-label">Phase 0</div>
    <div class="phase-title">Instrument &amp; unblock</div>
    <p>KPI 대시보드, 기준선 보고서, 아이덴티티 · 정책 블로커 제거.</p>
    <p class="phase-exit"><strong>Exit:</strong> 재현 가능한 기준선(reproducible baseline) 확보.</p>
  </li>
  <li>
    <span class="phase-marker" aria-hidden="true" />
    <div class="phase-label">Phase 1</div>
    <div class="phase-title">Rule catalog &amp; T0</div>
    <p>규칙 카탈로그 정규화, T0 정책 게이트 가동, remediation PR 자동 생성.</p>
    <p class="phase-exit"><strong>Exit:</strong> Change 게이트가 대상 스코프에서 shadow 모드로 운영.</p>
  </li>
  <li>
    <span class="phase-marker" aria-hidden="true" />
    <div class="phase-label">Phase 2</div>
    <div class="phase-title">Quality gate &amp; T1</div>
    <p>규칙 지속 업데이트, mixed-model 교차 검증 LLM 품질 게이트, 임베딩 기반 패턴 재사용(T1).</p>
    <p class="phase-exit"><strong>Exit:</strong> Phase 0 기준선 대비 자동해소율 검증.</p>
  </li>
  <li>
    <span class="phase-marker" aria-hidden="true" />
    <div class="phase-label">Phase 3</div>
    <div class="phase-title">Integrated autonomy</div>
    <p>통합 제어 루프, DR/Chaos 스케줄러(DB DR 훈련 포함), 리스크 게이트가 승인 흐름을 감독하는 FinOps 자동 액션.</p>
    <p class="phase-exit"><strong>Exit:</strong> 세 도메인 전반의 자율 MVP.</p>
  </li>
  <li class="future">
    <span class="phase-marker" aria-hidden="true" />
    <div class="phase-label">Phase 4 · TBD</div>
    <div class="phase-title">Scale</div>
    <p>지속 측정, 패턴 라이브러리 성장, 모델 비용/품질 추적, 확장성. 멀티 클라우드 확장은 TBD.</p>
  </li>
</ol>
